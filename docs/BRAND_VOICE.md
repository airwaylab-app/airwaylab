# AirwayLab Brand Voice Guide

> See also: `BRAND.md` (root) for full brand guidelines including microcopy rules,
> positioning, and commercial philosophy.

## Core Voice

AirwayLab sounds like a knowledgeable friend who happens to be really good with data.
Not a health tech startup. Not a medical device company. Not a generic open-source project.

### Voice Attributes

| Attribute | What it means | What it doesn't mean |
|-----------|--------------|---------------------|
| Empathetic | We understand the frustration of being "treated" but still tired | We're not soft or vague about the data |
| Direct | We state what the data shows clearly | We don't hedge everything with disclaimers (except where medically necessary) |
| Community-native | We sound like a knowledgeable forum member, not a corporation | We don't use marketing speak or startup jargon |
| Technically credible | We reference real methodologies (Glasgow Index, NED, WAT) | We don't gatekeep with jargon — plain language first, technical detail second |

### Tone by Context

| Context | Tone | Example |
|---------|------|---------|
| Landing page | Warm + confident | "Drag your SD card folder. Get a report in 30 seconds." |
| Analysis results | Precise + encouraging | "Your Glasgow Index averaged 3.2 this week — that's moderate flow limitation." |
| Error states | Honest + helpful | "We couldn't read this file format. AirwayLab currently supports ResMed EDF files." |
| Reddit/forums | Peer-to-peer + humble | "I built this because I was frustrated with the same thing." |
| GitHub README | Developer-friendly + clear | "Runs entirely client-side. No backend, no accounts, no data collection." |

### Writing Rules

1. Lead with the benefit, follow with the mechanism.
   YES: "See your flow limitation trends (using Glasgow Index breath scoring)"
   NO: "The Glasgow Index scores each breath on a 0-9 scale which we use to..."
2. Use "your" generously. "Your data", "your therapy", "your breathing patterns."
3. Say "PAP" not "CPAP" (inclusive of BiPAP, ASV users).
4. Avoid: "revolutionary", "cutting-edge", "AI-powered", "game-changing".
5. Medical disclaimers: end of analysis, small text, never leading. Never omit, but don't interrupt the flow.
6. Numbers always show the unit. "3.2 Glasgow" not "3.2" alone.
7. Don't say "we" in product UI copy. Use "AirwayLab" or passive voice. "We" is for community/personal contexts (Reddit, GitHub discussions).

### Messaging Hierarchy

**L1 — Mission:** "Your breathing data belongs to you."
**L2 — Value prop:** "See if your sleep therapy is actually working — beyond AHI."
**L3 — Differentiator:** "Flow limitation analysis, breathing pattern scoring, and real insights — all in your browser, nothing uploaded, 100% open source."
**L4 — Trust proof:** "Built by a PAP user. Powered by the Glasgow Index. Complements OSCAR. Your data never leaves your device."

### Competitor Positioning

**vs OSCAR:** "AirwayLab complements OSCAR — it's not a replacement. OSCAR shows you the raw waveforms. AirwayLab scores and interprets the patterns across nights."
**vs myAir:** AirwayLab is independent, deeper, and private. myAir is convenient and clinician-integrated.

### Words We Use / Words We Don't

| Use | Don't use |
|-----|-----------|
| PAP therapy | CPAP (unless specifically discussing CPAP-only) |
| flow limitation | airflow restriction |
| breathing patterns | respiratory events (too clinical) |
| your data | patient data |
| analysis | diagnosis (we don't diagnose) |
| insights | recommendations (too prescriptive) |
| open source | free software (GPL nuance aside, "open source" resonates better) |
